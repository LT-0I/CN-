# JSON 数据文件格式说明

本文档定义导学案课时数据的 JSON 格式规范。

---

## 文件结构

每个 JSON 文件是一个数组，每个元素代表一个课时：

```json
[
  { /* 课时1 */ },
  { /* 课时2 */ },
  ...
]
```

---

## 字段定义

```jsonc
{
  // 必填字段
  "totalLessonNum": 1,              // 整数，全册总课时编号（1-40）
  "unitLessonNum": "第1课时",       // 字符串，单元内课时标签
  "title": "面的旋转（一）",         // 字符串，课题名称
  "courseType": "新授课",            // 字符串，课型（新授课/练习课/复习课）
  "preparation": "课件、教具",       // 字符串，课前准备内容

  "objectives": {                    // 对象，三维学习目标
    "knowledge": "...",              // 字符串，知识与能力目标
    "process": "...",                // 字符串，过程与方法目标
    "emotion": "..."                 // 字符串，情感态度与价值观目标
  },

  "keyPoint": "...",                 // 字符串，教学重点
  "difficulty": "...",               // 字符串，教学难点

  "teaching": {                      // 对象，教学流程
    "introduction": "...",           // 字符串，一、激趣导入（100-150字）
    "mainTeaching": "...",           // 字符串，二、快乐导学（300-500字）
    "practice": "...",               // 字符串，三、学以致用（200-300字）
    "summary": "..."                 // 字符串，四、总结提升（50-100字）
  },

  "boardDesign": "..."               // 字符串，板书设计内容
}
```

---

## 格式规则

### 换行符
- 使用 `\n` 表示段落之间的换行
- 生成脚本会自动将 `\n` 拆分为独立段落

### 引号安全
- **禁止**在字符串值中使用未转义的双引号 `"`
- 中文引号用单引号 `''` 替代

```json
// 错误
"introduction": "今天学习\"面的旋转\"。"

// 正确
"introduction": "今天学习'面的旋转'。"
```

### 章节标题自动加粗
教学流程中以下开头的行会被自动识别为标题并加粗：
- `活动一：`、`活动二：` 等
- `一、`、`二、`、`三、`、`四、` 等

### 篇幅控制

| 字段 | 建议字数 | 说明 |
|------|----------|------|
| introduction | 100-150字 | 情境引入，简洁生动 |
| mainTeaching | 300-500字 | 核心教学，含具体步骤编号 |
| practice | 200-300字 | 2-4道练习题，含解答过程 |
| summary | 50-100字 | 课堂小结，回顾要点 |
| boardDesign | 5-10行 | 核心结构，用 `\n` 分行 |

## 文件命名规范

```
data/
├── unit1.json              # 第一单元
├── unit1_part1.json         # 第一单元第一部分（课时多时拆分）
├── unit1_part2.json         # 第一单元第二部分
├── unit2.json              # 第二单元
├── unit3.json
├── unit4.json
├── unit5_math_fun.json      # 特殊单元可用描述性名称
└── unit6_review.json        # 总复习
```
