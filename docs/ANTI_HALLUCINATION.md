# 防幻觉策略详解

本文档详细说明导学案生成过程中防止 AI 产生错误内容（幻觉）的策略和机制。

---

## 问题背景

AI 在生成大量教学内容时，容易出现以下错误：
1. **公式错误**：如圆锥体积写成 `1/2πr²h`（应为 `1/3πr²h`）
2. **计算错误**：如 `3.14×9×8=225.12`（正确为 `226.08`）
3. **概念混淆**：如把"正比例"和"反比例"的判断条件搞反
4. **事实错误**：如"莫比乌斯带沿中线剪开变成两个环"（应为一个扭转两次的大环）
5. **上下文遗忘**：在生成第30课时时，忘记前面课时中已建立的知识体系

---

## 策略一：分轮生成

### 原则
- 每轮最多生成 **1个单元**（6-8课时）
- 绝不一次性生成超过8个课时
- 每轮独立执行，通过 JSON 文件传递数据

### 为什么有效
- 减少单次生成的内容量，降低上下文遗忘的风险
- 每轮可以独立验证，发现错误只需重做该轮
- AI 在较短上下文中保持更高的准确性

### 实际效果
| 方式 | 课时数 | 错误风险 |
|------|--------|----------|
| 一次性生成40课时 | 40 | 极高（后期严重退化） |
| 分6轮，每轮6-8课时 | 6-8 | 低 |

---

## 策略二：知识点锚定清单

### 原则
每轮生成前，必须先提供该单元的 **核心知识点清单**，作为生成内容的"锚点"参照。

### 示例（圆柱与圆锥单元）
```
- 圆柱侧面积 = 底面周长 × 高 = πdh = 2πrh
- 圆柱表面积 = 侧面积 + 2×底面积 = 2πrh + 2πr²
- 圆柱体积 = 底面积 × 高 = πr²h
- 圆锥体积 = 1/3 × 底面积 × 高 = 1/3πr²h
- 圆柱与等底等高圆锥的体积关系：V柱 = 3 × V锥
```

### 为什么有效
- 提供了"真相源"，AI 在生成内容时有明确参照
- 减少凭记忆编造公式的可能性
- 清单中的精确表述会被直接引用到教案中

---

## 策略三：练习题自检验算

### 原则
所有包含数值计算的练习题，必须：
1. 完整写出求解步骤
2. 给出最终答案
3. 独立验算答案的正确性

### 示例
```
题目：一个圆柱底面半径5cm，高12cm，求体积。
解：V = πr²h = 3.14 × 5² × 12 = 3.14 × 25 × 12
验算：3.14 × 25 = 78.5，78.5 × 12 = 942
答：体积为 942 cm³ ✓
```

### 关键规则
- π 统一取 3.14，不取其他近似值
- 分步计算，不跳步
- 最终结果必须是精确数值（非约等于），除非题目要求保留小数

---

## 策略四：易错点主动标注

### 原则
在每轮的知识清单中，显式标注已知的常见错误点。

### 示例清单
| 易错点 | 正确表述 |
|--------|----------|
| 圆锥体积是圆柱的1/3 | 必须在"等底等高"前提下 |
| 正方形面积与边长成正比例 | 错！S/a=a 不是定值 |
| 莫比乌斯带沿中线剪开变两个环 | 错！变成一个扭转两次的大环 |
| 任意三角形旋转成圆锥 | 错！必须是直角三角形绕直角边 |

---

## 策略五：数据与逻辑分离

### 原则
课时内容数据（JSON）与文档生成脚本（JS）完全分离。

### 为什么有效
- JSON 数据可以独立审查和修正
- 发现某课时内容有误，只需修改对应 JSON 字段，不影响其他课时
- 生成脚本是确定性的（相同输入必然相同输出），不引入随机性

### 文件结构
```
data/
├── unit1.json     ← 可独立审查
├── unit2.json
└── ...
scripts/
└── generate.js    ← 纯格式逻辑，不含教学内容
```

---

## 策略六：JSON 安全

### 常见问题
```json
// 错误：字符串内包含未转义的双引号
"introduction": "今天学习"面的旋转"。"

// 正确：用单引号替代
"introduction": "今天学习'面的旋转'。"
```

### 检查方法
生成 JSON 后，在 Node.js 中测试：
```javascript
JSON.parse(fs.readFileSync('data/unit1.json', 'utf-8'));
// 如果不抛异常则安全
```

## 实施检查清单

生成每轮内容时，逐项检查：

- [ ] 本轮课时数 ≤ 8
- [ ] 已列出核心知识锚定清单
- [ ] 清单中标注了已知易错点
- [ ] 所有练习题答案已验算
- [ ] JSON 文件语法正确（无未转义引号）
- [ ] 运行脚本生成 docx 成功
- [ ] 学习目标包含三个维度
- [ ] 教学流程包含四个环节（激趣导入/快乐导学/学以致用/总结提升）
