# 分轮提示词模板

本文档提供分轮生成导学案的提示词模板。用户只需替换 `[方括号]` 中的内容即可使用。

---

## 【第0轮：模板解析与基础设施搭建】

```
你是一位中国[省份][城市][区县]的资深小学[科目]教师。请严格按照我提供的模板文件（[模板文件名].docx）的格式、排版和表格结构，为[教材版本]小学[科目][年级][学期]生成完整的全册教师导学案Word文档（.docx）。

本任务内容量大（全册约[N]课时），将分多轮对话完成。本轮是第0轮，任务是：解析模板 + 搭建生成脚本框架 + 生成第一单元作为验证。

## 一、模板说明

模板文件路径：[模板文件路径]

请先解包并仔细分析此模板的完整结构，包括：
- 页面尺寸、页边距、字体、字号
- 表格的列宽、行高、边框样式、单元格合并方式
- 每课时的页面数量（约[N]页/课时）
- 页眉页脚设置

## 二、内容质量要求

1. **课标对齐**：所有教学目标须符合《义务教育[科目]课程标准（2022年版）》
2. **三维目标**：每课时均须包含"知识与能力""过程与方法""情感态度与价值观"
3. **教学流程**：
   - "激趣导入"须创设贴近学生生活的情境（可结合本地元素）
   - "快乐导学"须包含具体的师生互动步骤
   - "学以致用"须包含层次分明的练习题
   - "总结提升"须引导学生回顾核心知识
4. **板书设计**：须体现核心知识的逻辑结构

## 三、本轮核心知识锚定清单

[列出第一单元的核心知识点、公式、定理等]

**自检要求**：[针对本单元的具体自检要求]

## 四、本轮课时列表（总课时编号1-[N]）

[列出第一单元所有课时的编号、名称和简要描述]

## 五、执行步骤

1. 读取 docx skill
2. 解包模板文件，分析格式参数
3. 创建 generate.js 脚本框架
4. 创建第一单元 JSON 数据文件
5. 运行脚本生成 docx
6. 验证输出

完成后请告知第一单元的生成结果。
```

## 【第N轮：后续单元模板】

```
继续生成[科目]导学案。本轮是第[N]轮，生成第[X]单元。

## 本轮核心知识锚定清单

[列出本单元的核心知识点]

**自检要求**：[本单元的自检要求]

## 本轮课时列表（总课时编号[M]-[M+K]）

[列出本单元所有课时]

## 执行步骤

1. 创建 data/unit[X].json
2. 运行 node generate.js output/unit[X].docx data/unit[X].json [--compact]
3. 验证输出
```

```

## 执行步骤

1. 确认以下文件均已生成：
   [列出所有单元文件]

2. 合并命令：
   node generate.js output/[科目]导学案（全册）.docx data/unit1.json data/unit2.json ...

3. 验证合并后的文档
```

## 适配提示

| 需要替换的内容 | 说明 | 示例 |
|---|---|---|
| `[省份][城市][区县]` | 学校所在地 | 山西省晋城市泽州县 |
| `[科目]` | 教学科目 | 数学 |
| `[年级]` | 年级 | 六年级 |
| `[学期]` | 上/下册 | 下册 |
| `[教材版本]` | 出版社版本 | 北师大版 |
| `[模板文件名]` | 模板文件 | 六上数学 |
| `[N]` | 总课时数 | 40 |
| 核心知识锚定清单 | 每轮必须提供 | 见SKILL.md示例 |
