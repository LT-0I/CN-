# Teaching Plan Generator Skill

## 概述

本 Skill 用于根据 Word 导学案模板，自动生成指定科目、学期、教材版本的全册教师导学案。整个流程分为 **模板解析 → 脚本搭建 → 分轮数据生成 → 文档输出 → 合并全册** 五个阶段。

---

## 前置条件

1. 用户提供一份 `.docx` 模板文件（已有的导学案样例）
2. 用户指定：科目、年级、学期（上/下册）、教材版本
3. 环境已安装 `Node.js` 和 `docx` 库（`npm install -g docx`）
4. 环境已安装 `Python`（用于 unpack/validate 脚本）

---

## 执行流程（严格按顺序）

### 阶段一：模板解析（必须先做）

```
步骤 1.1：读取 docx skill（调用 document-skills:docx）
步骤 1.2：解包模板文件
  python scripts/office/unpack.py <模板.docx> unpacked_template/
步骤 1.3：分析 unpacked_template/word/document.xml，提取以下参数：
  - 页面尺寸（width/height，单位 DXA）
  - 页边距（top/bottom/left/right）
  - 表格列宽、表格缩进、边框样式
  - 字体（通常为宋体）、字号（通常 28=14pt）
  - 表格行结构（哪些行有 gridSpan 合并、哪些列 vMerge 纵向合并）
  - 段落缩进（firstLine 值）、行距（line 值）
  - 页眉内容
步骤 1.4：将解析结果记录（硬编码入生成脚本的常量区）
```

**关键原则：**
- 不要猜测格式，一切参数从模板 XML 中实际读取
- 特别注意表格的 gridSpan（跨列合并）和 vMerge（跨行合并）
- 注意"导学流程"列的竖排文字（每个字一个段落）

### 阶段二：脚本搭建

```
步骤 2.1：创建 generate.js 脚本，结构如下：
  - 常量区：页面参数、列宽、边框样式（来自阶段一）
  - 辅助函数：boldRun(), normalRun(), songRun(), createCell()
  - 表格构建函数：createInfoTable(), createTeachingTable(), createBoardTable()
  - 课时组装函数：createLessonSections(lesson, isFirst, compact)
  - 主函数：generate(dataFiles, outputPath)
步骤 2.2：支持 --compact 参数（2页/课时，去标题页眉，缩小页边距）
步骤 2.3：数据格式为 JSON 数组，每个元素代表一个课时
```

**JSON 数据格式（必须严格遵守）：**
```json
{
  "totalLessonNum": 1,
  "unitLessonNum": "第1课时",
  "title": "课题名称",
  "courseType": "新授课",
  "preparation": "课件、教具",
  "objectives": {
    "knowledge": "知识与能力目标",
    "process": "过程与方法目标",
    "emotion": "情感态度与价值观目标"
  },
  "keyPoint": "重点内容",
  "difficulty": "难点内容",
  "teaching": {
    "introduction": "一、激趣导入（100-150字）",
    "mainTeaching": "二、快乐导学（300-500字，用\\n分段）",
    "practice": "三、学以致用（200-300字，含2-4道练习题）",
    "summary": "四、总结提升（50-100字）"
  },
  "boardDesign": "板书设计内容（用\\n分行）"
}
```

### 阶段三：分轮数据生成（核心防幻觉阶段）

**必须分轮生成，每轮最多处理一个单元（6-8课时）。**

```
每轮执行步骤：
步骤 3.1：确定本轮单元名称和课时列表
步骤 3.2：编写本轮的「核心公式与事实核查清单」
步骤 3.3：生成 JSON 数据文件
步骤 3.4：对 JSON 中的每道练习题进行答案验算
步骤 3.5：运行 generate.js 生成本轮 docx
步骤 3.6：验证输出文件
步骤 3.7：记录进度
```

**分轮策略规则：**

| 规则 | 说明 |
|------|------|
| 每轮上限 | 一个单元（6-8课时），不超过8课时 |
| 知识点锚定 | 每轮开头列出该单元所有核心公式/定义/定理 |
| 自检验算 | 所有练习题答案必须自行验算（如 π≈3.14 的计算） |
| 易错标注 | 标注已知的常见错误点（如"莫比乌斯带沿中线剪开不是变成两个环"） |
| JSON 校验 | 生成 JSON 后检查是否有未转义引号等语法错误 |
| 内容去引号 | JSON 字符串内的中文引号用单引号替代，避免 JSON 解析错误 |

### 阶段四：文档输出与验证

```
步骤 4.1：运行 node generate.js output/<单元>.docx data/<单元>.json [--compact]
步骤 4.2：（可选）用 validate.py 验证
步骤 4.3：（可选）解包输出文件检查 XML 结构
```

### 阶段五：合并全册

```
步骤 5.1：确认所有单元 docx 均已生成
步骤 5.2：运行合并命令：
  node generate.js output/全册导学案.docx data/unit1.json data/unit2.json ... [--compact]
步骤 5.3：验证合并后文件
```

## 防幻觉策略（必须遵守）

### 1. 分轮生成
- **绝对不允许**一次性生成超过 8 个课时的内容
- 每轮独立处理一个单元，通过 JSON 文件传递数据

### 2. 知识点锚定清单
每轮生成前，必须先列出该单元的核心知识清单，作为内容生成的"锚点"。

**示例（圆柱与圆锥单元）：**
```
- 圆柱侧面积 = 底面周长 × 高 = 2πrh
- 圆柱表面积 = 2πrh + 2πr²
- 圆柱体积 = πr²h
- 圆锥体积 = 1/3πr²h
- 圆柱与等底等高圆锥：V柱 = 3 × V锥
```

### 3. 练习题自检
- 每道包含数值计算的练习题，必须完整写出求解过程和答案
- 涉及 π 的计算统一取 π≈3.14，并验算最终数值
- 验算方法：独立重新计算一遍，对比结果

### 4. 易错点主动标注
- 在知识清单中标注已知的易错点
- 例：`正方形面积与边长不成正比例（S=a²，S/a=a 不是定值）`
- 例：`莫比乌斯带沿中线剪开 → 一个扭转两次的大环（不是两个环）`

### 5. JSON 安全
- JSON 字符串内不使用未转义的双引号
- 中文引号（""）在 JSON 中用单引号（''）替代
- 生成 JSON 后先用 `JSON.parse()` 测试是否可解析

### 6. 教学内容质量要求
- 学习目标必须包含三个维度：知识与能力、过程与方法、情感态度与价值观
- 激趣导入须贴近学生生活，可融入本地元素
- 快乐导学须有具体的师生互动步骤和编号
- 学以致用须有层次分明的练习题（基础→提高）
- 板书设计须体现核心知识的逻辑结构

## 适配不同科目的要点

### 数学
- 核心锚定：公式、定理、计算规则
- 自检重点：练习题数值计算验算
- 易错点：公式混淆、单位换算、条件前提（如等底等高）

### 语文
- 核心锚定：生字词表、课文主旨、修辞手法
- 自检重点：字词读音和书写的准确性
- 易错点：多音字、形近字、文言文翻译

### 英语
- 核心锚定：单词表、语法点、句型结构
- 自检重点：拼写、语法规则的准确性
- 易错点：时态混淆、不规则变化

### 科学
- 核心锚定：实验步骤、科学原理、观察结论
- 自检重点：实验结论的科学准确性
- 易错点：因果关系、对照实验的控制变量

## 使用示例

**用户输入：**
> 请按照"六上数学.docx"模板，生成北师大版小学数学六年级下册的教师导学案。

**AI 执行流程：**
1. 调用 `document-skills:docx` skill 读取模板
2. 解包模板文件，提取格式参数
3. 创建 generate.js 脚本
4. 确定课时列表（共40课时，6个单元）
5. 分6轮生成，每轮一个单元
6. 合并全册输出

---

## 注意事项

- "主备教师""审核教师""导学反思""二度备"栏均留空，供教师手写
- 课时之间用分页符（Section）分隔
- 如用户要求调整页数（如从3页改为2页），通过 `--compact` 参数实现
- 如某轮出错，只需重新生成该轮 JSON 数据文件并重新运行脚本
